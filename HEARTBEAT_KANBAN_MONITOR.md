# 📊 Kanban Board Heartbeat Monitor

## Overview

Automatic Kanban Board monitoring system that checks for updates and sends Discord notifications when changes are detected.

---

## 📁 Files Created

| File | Purpose |
|------|---------|
| `HEARTBEAT.md` | Updated with Kanban monitoring task |
| `memory/kanban-last-checked.json` | State tracking file |
| `scripts/check-kanban-updates.js` | Monitoring script |
| `HEARTBEAT_KANBAN_MONITOR.md` | This documentation |

---

## 🔧 How It Works

### Every Heartbeat Check

1. **Read** `kanban-board.json` metadata
2. **Compare** `meta.updated` timestamp with last checked state
3. **If changed:**
   - Analyze what changed (new projects, completed, status changes)
   - Generate Discord notification message
   - Send to `#kanban-updates` channel
   - Update state file
4. **If unchanged:** Skip notification

### State Tracking

**File:** `memory/kanban-last-checked.json`

```json
{
  "lastChecked": "2026-03-01T14:45:00Z",
  "lastUpdated": "2026-03-01T09:30:00Z",
  "projectCount": 19,
  "completedProjects": 13,
  "inProgressProjects": 4,
  "todoProjects": 1,
  "blockedProjects": 1
}
```

## 🎯 Detection Logic

### Changes Detected

1. **New Projects** - Project count increased
2. **Completed Projects** - Projects moved to "done" status
3. **Status Changes** - Distribution changes across columns
4. **Metadata Updates** - Any change to `meta.updated`

### No Notification When

- `meta.updated` unchanged since last check
- During sleep time (23:00-08:00 HKT) unless urgent
- First run (creates baseline state)

## 🔧 Customization

### Change Discord Channel

Edit `scripts/check-kanban-updates.js`:
```javascript
const DISCORD_CHANNEL_ID = 'YOUR_CHANNEL_ID';
```

### Adjust Check Frequency

Modify in `HEARTBEAT.md` - heartbeat runs automatically based on your OpenClaw configuration.

### Notification Filters

Add filters in the script to only notify for:
- High priority projects
- Specific status changes
- Minimum time between notifications

---

## 🐛 Troubleshooting

### Not Detecting Changes

1. Check `meta.updated` is actually changing in `kanban-board.json`
2. Verify state file exists: `memory/kanban-last-checked.json`
3. Run with `--force` flag to test

### Sending Too Many Notifications

1. Add debounce logic to script
2. Increase heartbeat interval
3. Add filters for significant changes only

### State File Corrupted

```bash
rm memory/kanban-last-checked.json
# Will create fresh state on next run
```
